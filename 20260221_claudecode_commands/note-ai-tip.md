# /note-ai-tip — AI活用ティップス記事ジェネレーター

「えーあいと生きる練習中。」アカウント向けに、会話ログから
AI活用の発見・おすすめ・事例を「読者への提案」スタイルで記事化する。

## 手順

### 1. プロジェクトハッシュの特定

```bash
pwd | sed 's|/|-|g' | sed 's|^-||'
```

### 2. 最新JONLファイルの特定

```bash
ls -t ~/.claude/projects/{ハッシュ}/*.jsonl 2>/dev/null | grep -v subagents | head -1
```

ファイルが見つからない場合は「会話ログが見つかりません」と報告して終了。

### 3. 全発言テキストを時系列で抽出

```bash
python3 -c "
import json
messages = []
entries = []
for line in open('{JONLパス}'):
    try:
        entries.append(json.loads(line))
    except: pass

for obj in entries:
    role = obj.get('type', '')
    if role not in ('user', 'assistant'):
        continue
    for block in obj.get('message', {}).get('content', []):
        if isinstance(block, dict) and block.get('type') == 'text':
            t = block['text'].strip()
            if (len(t) > 30
                and not t.startswith('{')
                and not t.startswith('#')
                and not t.startswith('<')):
                messages.append(f'[{role.upper()}] {t[:800]}')

print('\n\n'.join(messages))
"
```

### 4. 記事の生成

抽出した全発言テキストを読み込み、以下の方針で記事を作成する。

**テーマの確認:**
引数でAI活用のテーマが指定されている場合はそれを使う。
指定がない場合は、会話から「読者が試せるAIの使い方」として最も具体的・再現性のある発見を1つ選ぶ。

**分析フェーズ（内部処理）:**
- 会話の中で「AIをこう使ったら○○だった」という発見を特定する
- その発見の再現手順・コツを抽出する
- 読者が「やってみよう」と思える具体性があるか確認する
- 読者へのメリット・気づきを言語化する

**執筆方針:**
- 「えーあいと生きる練習中。」アカウントの発見・体験として書く
- 読者への提案・おすすめが主軸。思考の旅ではなく「こう使うといいよ」が骨格
- 自分の体験・具体例を根拠として使う
- 試し方・再現方法を必ず含める（読者が真似できる形で）
- 対話形式は使わない。一人称で書く
- 文体：だ・である調（断定的でテンポよく）
- 長さ：700〜1200字程度

**構成:**
```
リード：こういう使い方を発見した・試した（2〜3文）
↓
体験・具体例：自分はこうやった
↓
なぜ効くのか：仕組み・理由
↓
試し方：読者がすぐ再現できる手順やコツ
↓
締め：「やってみて」で終わる（余韻より背中を押す）
```

**フォーマット:**
```markdown
---
title: {タイトル（「〇〇という使い方」「AIで〇〇できた」など具体的に）}
date: {今日の日付 YYYY-MM-DD}
tags: [AI, AIとやってみた, {テーマに応じたタグ}]
---

# {タイトル}

{リード：発見・提案を一言で、2〜3文}

---

## {体験セクション見出し}

{具体的な自分の体験}

---

## なぜ効くのか

{仕組み・理由}

> {印象的なフレーズ}

---

## 試し方

{再現手順・コツ（箇条書きでもよい）}

---

{締め：読者の背中を押す一文}
```

### 5. ファイルへの書き込み

- 出力先: `{カレントディレクトリ}/articles/`（なければ作成）
- ファイル名: `{YYYYMMDD}_{タイトルをスネークケース}.md`
- 日付は今日の日付を使用

### 6. 完了報告

生成したファイルのパスと、記事のタイトル・文字数を報告する。
